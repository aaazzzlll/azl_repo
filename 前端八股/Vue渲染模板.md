在 Vue 中，默认情况下模板中的 HTML 注释会在渲染过程中被移除，不会出现在最终的 DOM 中。这是因为 Vue 的模板编译器会将注释视为无用内容并自动删除，以减小最终生成的代码体积。

保留注释方法：
	1. 使用comments选项（Vue 2）：在 Vue 2 中，可以通过在组件选项中设置 `comments: true` 来保留模板中的 HTML 注释
	2. 使用v-pre（Vue 2/Vue 3通用）：Vue 的免编译指令，加了之后元素内的 Vue 语法（{{}}、v-xxx）都不解析，直接显示原文
	3. 使用特殊注释语法（Vue 3）：在 Vue 3 中，可以使用特殊的注释语法 `<!--[COMMENTS]-->` 来保留注释


## Vue模板编译过程
	1. 解析：把模板字符串拆成AST（抽象语法树），让Vue看懂模板的结构和逻辑
	2. 转换/优化：优化AST并处理指令逻辑，提升渲染性能
	3. 生成：把AST转成渲染函数，最终让浏览器能执行并渲染出页面（根据数据生成虚拟DOM,最终渲染成真实DOM）

**数据变 → 生成新 VNode → 对比新旧 VNode 找差异 → 只更新真实 DOM 的差异部分 —— 这就是 Vue “高效更新页面” 的核心逻辑**


服务端渲染（[SSR](SSR.md)）中的注释：
在服务端渲染中，注释的处理可能与客户端渲染有所不同。通常，SSR 输出的 HTML 会包含注释，但在客户端激活（hydration）过程中，这些注释可能会被移除。
