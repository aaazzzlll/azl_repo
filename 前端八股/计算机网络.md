DNS(Domain Name System)：域名系统

# 传输层

- 传输层任务：为运行在不同主机上的应用进程提供直接的通信服务

- TCP/IP体系的传输层使用**端口号**（16bit表示，取值范围0~65535）来区分应用层的不同应用进程
	端口号只具有本地意义，只是为了标识本计算机应用层中的各进程，在因特网中，不同计算机中的相同端口号没有联系
	
	TCP/IP体系的应用层常用协议所使用的运输层熟知端口号：
	UDP（协议字段17）: RIP 520
		DNS 53
		TFTP 69
		SNMP 161
		DHCP 67/68
	TCP（协议字段6）: SMTP 25
		FTP 21/20
		BGP 179
		HTTP 80
		HTTPS 443

- 传输层协议又称为端到端协议
	因特网的传输层为应用层提供了2种不同的传输协议：面向连接的TCP、无连接的UDP

UDP和TCP的对比：

| UDP                         | TCP                        |
| --------------------------- | -------------------------- |
| 无连接                         | 面向连接                       |
| 支持一对一，一对多，多对一和多对多交互通信       | 每一条TCP连接只能有两个端点EP,只能是一对一通信 |
| 对应用层交付的报文直接打包               | 面向字节流                      |
| 尽最大努力交付，也就是不可靠；不使用流量控制和拥塞控制 | 可靠传输，使用流量控制和拥塞控制           |
| 首部开销小，仅8字节                  | 首部最小20字节，最大60字节            |
- TCP的流量控制(让发送方的发送速率不要太快，要让接收方来得及接收)
	利用[滑动窗口]机制实现：TCP接收方利用自己的接收窗口的大小来限制发送方发送窗口的大小
		TCP发送方收到接收方的[零窗口通知]后，应启动持续计时器。持续计时器超时后，向接收方发送[零窗口探测报文]

- TCP的拥塞控制
	拥塞：在某段时间，若对网络中某一资源的需求超过了该资源所能提供的可用部分，网络性能就要变坏。

- TCP的运输连接管理(使运输连接的建立和释放都能正常进行)
	TCP运输连接有以下三个阶段：
		建立TCP连接(三报文握手)
		数据传送
		释放TCP连接(四报文挥手)
