1. 连接
	HTTP/1.0默认使用非持久连接，每次请求都要新建一个TCP连接,但是可以
	HTTP/1.1默认使用持久连接，同一个TCP连接可以复用

2. 管道化支持
	HTTP/1.0不支持
	HTTP/1.1支持连续发送多个请求而不等待响应

3. 缓存
	HTTP/1.0的缓存规则比较粗糙
	HTTP/1.1引入更精细的缓存策略

4. host头
	HTTP/1.0没有host头
	HTTP/1.1强制要求host头字段，HTTP/1.1 通过 Host 头告诉服务器你要访问哪个，这就意味着一台服务器可以托管多个网站

5. 资源请求
	HTTP/1.0不支持请求资源的一部分，也不支持断点续传
	HTTP/1.1支持请求资源的一部分，也支持断点续传

6. 请求方法
	HTTP/1.1对请求方法进行了扩展和标准化，方法集比HTTP/1.0更完整规范
