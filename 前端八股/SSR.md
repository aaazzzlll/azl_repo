SSR（Server-Side Rendering，服务端渲染）是 **一种网页渲染方式**：页面的 HTML 结构不是在浏览器端（客户端）通过 JS 动态生成，而是在服务器端提前渲染好完整的 HTML，再将整个 HTML 页面发送给浏览器。

#### 核心作用

- **提升首屏加载速度**：尤其对慢网络 / 低端设备，无需等待 JS 加载执行就能看到页面内容；
- **优化 SEO**：搜索引擎爬虫能直接读取服务器返回的完整 HTML 内容（CSR 页面爬虫可能只拿到空骨架，无法解析动态内容）；
- **提升用户体验**：首屏无 “白屏”，用户能更快看到内容。


SSR 返回的 HTML 是静态的，浏览器加载 JS 后会 “激活” DOM，让页面具备交互能力（如点击事件），这个过程叫水合(hydration)
