渲染进程是多线程，主要包含以下7个线程：
1. **主线程**（GUI渲染和JS引擎共存的线程）
	这是渲染进程中最忙、最重要的线程：
		*解析和渲染页面（GUI渲染）*：读 HTML → 构建 DOM 树 → 读 CSS → 构建样式规则 → 组合成渲染树 → 计算布局（Layout / Reflow） → 绘制（Paint） → 生成待绘制的指令列表。
		*执行JavaScript*：解析JS代码，运行JS（V8引擎就在这里工作）
	关键限制：JS执行时，页面渲染会暂停；渲染更新时，JS也得等。因此如果JS写得很耗时（死循环、大量计算），页面就会卡住、假死、白屏很久
2. **定时器线程**
	专门跑setTimeOut和setInterval的独立线程，如果把定时器放在主线程，JS一阻塞，定时器就不准了
	计时结束 → 通过事件触发线程把回调任务放入任务队列 → 等主线程空闲执行
3. **事件触发线程**
	负责监听各种事件（定时器触发、异步请求完成、鼠标点击、键盘输入、页面滚动等），当事件发生时，把对应的事件回调任务放入任务队列，等主线程空闲时执行
4. **异步http请求线程**
	- 当页面 JS 执行 XMLHttpRequest / fetch 时，渲染进程的主线程把请求参数交给这个线程。
	- 这个线程通过 IPC 把请求“转发”给浏览器进程的网络进程。
	- 网络进程完成请求后，把响应数据通过 IPC 发回渲染进程。
	- 这个线程检测到响应到达（或状态变化），通过事件触发线程把对应的回调任务放入**任务队列**，等待主线程（JS 引擎）空闲时执行
5. **合成线程**
	把主线程提前准备好的“图层列表”在 GPU 上快速组合（合成）、变换、滚动、做简单动画，然后交给 GPU 进程显示
6. **I/O线程**
	用来和其他进程进行通信
