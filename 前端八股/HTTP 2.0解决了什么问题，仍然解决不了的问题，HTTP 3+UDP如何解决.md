HTTP/2.0解决的问题：一个连接能并行处理多个请求，优化了传输效率
	多路复用：单连接多路复用，省资源
	应用层队头阻塞：各个数据流独立，互不阻塞
	头部压缩：HPACK压缩，省带宽
	服务器主动推送
	解析效率高：二进制分帧，机器处理快

HTTP/2.0仍解决不了的问题：
	TCP层队头阻塞：HTTP/2.0所有流共享一个TCP连接，如果某个流的数据包丢失，TCP要求重传，后续所有流的数据即使到了也必须等待

HTTP/3+UDP：用QUIC替代TCP，基于UDP实现可靠传输
	流真正独立：每个流有独立的流ID和流量控制，丢包只在流内重传，不影响其他流，解决了TCP层的队头阻塞
	内置TLS 1.3：HTTP/2需要TCP三次握手和TLS握手，HTTP/3握手和加密合并，
