# 异步方案
**让耗时操作不阻塞主线程，等操作完成后再执行指定逻辑的实现方式**

## 回调函数
异步编程的最基本方法，将异步操作完成后要执行的逻辑，封装成函数作为参数，传给异步方法，异步操作完成后自动调用这个函数
	优点：简单、易理解和实现
	缺点：多次调用会形成回调地狱


## 事件监听
异步任务的执行不取决于代码的执行顺序，而取决于某个事件是否发生
	优点：易理解，对于每个事件可以指定多个回调函数，而且可以去耦合，利于实现模块化
	缺点：整个程序都变成事件驱动型，运行流程不清晰
