**私有属性和方法**：只能在类/对象的内部访问，外部无法读取、修改或调用的属性/方法

1. *从命名上规定*：在属性/方法名前面加_   表示是私有属性/方法
	这种方式只是一种命名规范，告诉开发者这是私有属性/方法，如果想要调用也是可以调用的

2. *Proxy*
	Proxy 可以定义目标对象的 get、set、Object.keys 的逻辑，可以在这一层做一下判断，如果是下划线 _ 开头就不让访问，否则就可以访问。
3. *Symbol*
	使用Symbol生成唯一的值作为属性名，外部无法直接访问这类属性。但是使用 Object.getOwnPropertySymbols，可以取到对象的所有 Symbols 属性，然后就可以拿到属性值
	这种方式只是 Object.keys 取不到对应的属性而已，不如 Proxy 那种方式完善
4. *WeakMap*
	给每个属性定义一个WeakMap来维护，key为当前对象，value为该对象对应属性的属性值
5. *`#prop`*：这是 ES2022 标准化的私有属性 / 方法写法，也是目前最推荐的方案，浏览器 / Node.js 已全面支持
	用 `#` 前缀定义的属性 / 方法，只能在类内部访问
6. *ts 的private*：编译时约束，运行时无限制
	ts 可以通过 private 来修饰属性、方法的可见性：
	- private 表示属性私有，只有 class 内部可访问
	- protected 表示保护，只有 class 和子 class 可访问
	- public 表示公有，外部也可访问
