HTTP 1.1到2.0：解决队头阻塞和带宽问题

1. 多路复用：
	HTTP/1.1的一个连接上必须等前一个请求-响应完成，才能发下一个请求（请求串行）
	HTTP/2.0将每个请求/响应拆分成多个帧，不同请求的帧可以交错发送，到达后再组装。多个流完全独立，互不阻塞
2. 头部压缩
	HTTP/1.1每次请求都携带完整的文本头部，大量重复数据浪费带宽
	HTTP/2.0实现了头信息压缩。一方面，**使用HPACK算法压缩头部**；另一方面，客户端和服务器同时维护一张**头信息表**，所有字段都会存入这个表，生成一个索引号，以后就只发送索引号，可以提高速度
3. 服务器推送
	HTTP/1.1只能客户端主动请求，服务器被动响应
	HTTP/2.0服务器可以主动推送资源
