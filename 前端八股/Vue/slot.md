基本概念：插槽是组件内的一个占位符，让我们可以向组件内部插入内容。父组件可以在使用子组件时，在子组件标签内传入内容，这些内容会被渲染到子组件内部的slot标签位置

插槽类型：
	1. 默认插槽：没有名字的插槽，每个组件只能有一个
	2. 具名插槽：带有name属性的插槽，可以有多个。使用template和v-slot指令来指定内容应该插入到哪个具名插槽
	3. 作用域插槽：能够接收子组件传递的数据的插槽

### 作用域插槽
```html
//子组件传递数据
<slot :text="message" :count="1"></slot>
```

```html
//父组件接收数据
<MyComponent v-slot="slotProps">
  {{ slotProps.text }} {{ slotProps.count }}
</MyComponent>
```

插槽特点：
	1. 插槽内容可以访问到父组件的数据作用域
	2. 插槽内容无法访问子组件的数据作用域
	3. slot标签可以包含默认内容，当父组件没有提供内容时显示

插槽的应用场景：
	1. 内容分发：允许父组件向子组件的指定位置传递任意内容（文字、标签、组件），解决子组件内容固定的问题
	2. 组件复用：基于插槽定制通用组件的差异化内容，让同一个基础组件适配多个业务场景（如列表、弹窗），减少重复代码
	3. 组件封装：通过插槽预留扩展点，让封装的组件既保留核心功能，又能灵活适配不同业务需求，平衡封装性和灵活性
