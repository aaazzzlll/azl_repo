- *实现路径前缀 + 角色校验的权限管控方案，封装 access.ts 独立模块集中处理路由拦截与跳转逻辑，结合菜单过滤实现前后端一致的权限体验。*


access.ts文件核心逻辑
```text
用户访问任意页面
       │
       ▼
  beforeEach 触发
       │
       ├─ 首次加载？──是──→ await 请求后端获取用户信息 → 标记 firstFetch = false
       │                                                        │
       ├────────────────────────────────────────────────────────┘
       │
       ▼
  目标路径以 /admin 开头？
       │
       ├─ 否 ──→ next() 放行
       │
       └─ 是 ──→ 用户是 admin？
                   │
                   ├─ 是 ──→ next() 放行
                   │
                   └─ 否 ──→ message.error('没有权限')
                             next('/user/login?redirect=原地址')
                             return（终止）
```

1. **前端权限控制是怎么实现的**
	我做了两层控制：
		==菜单级别==
			在封装的GlobalHeader组件中用computed根据Pinia中的用户角色动态过滤菜单项，非管理员看不到需要管理员身份的菜单项
		==路由级别==
			在access.ts中注册router.beforeEach全局前置守卫，判断路径是否是以/admin开头，如果是且用户不是管理员，就阻止导航并重定向到登录页，即使用户手动输入URL，也无法访问管理页面
