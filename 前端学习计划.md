### 阶段一：框架速成（第 1-2 周）

**目标：从“写玩具”进阶到“写工程”。** 现在的公司不会让你用原生 JS 写项目。你必须掌握一个主流框架（Vue 或 React）。

- **选型建议：** 如果你追求上手快、国内岗位多，选 **Vue 3**；如果你想去外企或字节/阿里等大厂，选 **React**。如果不确定，建议先学 **Vue 3**（因为它对新手更友好，概念更少）。
    
- **学习重点（不要全学，只学面试用的）：**
    
    - **Vue:** 响应式原理 (ref/reactive)、生命周期 (onMounted)、组件通信 (props/emit)、Pinia (状态管理)、Vue Router。
        
    - **React:** Hooks (useState, useEffect 是核心)、Redux Toolkit、React Router。
        
- **产出要求：** 跟着视频教程做一个简单的“后台管理系统”或“移动端商城”Demo。**不要只看，一定要手敲代码！**
    

### 阶段二：打造“简历级”项目（第 3-5 周）

**目标：做一个能写在简历上、能讲出亮点的项目。**

- **千万不要做：** 简单的 To-Do List、计算器、静态博客。这些烂大街了。
    
- **要做这种：**
    
    1. **仿站类：** 仿网易云音乐、仿知乎日报（要有登录、列表、详情、搜索）。
        
    2. **工具类：** 可视化大屏（用 ECharts）、组件库（自己封装几个简单组件，如 Button, Input）。
        
- **关键指标（面试官必问）：**
    
    - 要有 **API 请求**（用 Axios 调接口）。
        
    - 要有 **状态管理**（登录态怎么存？）。
        
    - 要有 **路由跳转**。
        
    - **加分项：** 做了“防抖节流”优化（把你刚背的代码用进去！）、做了“懒加载”。
        

### 阶段三：八股文与算法穿插（贯穿全程，第 1-7 周）

不要专门留时间背书，要把这些分散在每一天。

#### 1. 算法（每天 1 小时，不能停）

**策略：** 利用我刚才给你的“模版法”，只做 **LeetCode Hot 100**。

- **重点：** 数组、链表、树、哈希表。
    
- **心态：** 即使是实习，算法也是敲门砖，不需要做很难，但简单题必须秒杀。
    

#### 2. 前端八股文（每天 1 小时）

实习面试非常看重基础。以下是**必考题**，必须背得滚瓜烂熟：

- **JS 核心：** 原型链、闭包、`this` 指向、Promise（尝试手写一个简单的 Promise）、Event Loop（宏任务/微任务）。
    
- **CSS：** Flex 布局（必考）、BFC、水平垂直居中、移动端适配（rem/vw）。
    
- **网络：** HTTP 状态码、GET/POST 区别、跨域怎么解决、浏览器缓存。
    
- **浏览器：** 从输入 URL 到页面显示发生了什么（经典中的经典）。
    

### 阶段四：简历与模拟面试（第 6-7 周）

**目标：包装自己，模拟实战。**

- **简历优化：**
    
    - 不要写“精通 JavaScript”，写“熟悉 JavaScript 核心特性（闭包、原型链）”。
        
    - 项目经历用 **STAR 法则** 写：遇到了什么难点 -> 用了什么技术 -> 解决了什么问题。
        
- **手写题突击：**
    
    - 把你之前背的 **防抖、节流、深拷贝、数组去重、数组扁平化** 再默写几遍。
