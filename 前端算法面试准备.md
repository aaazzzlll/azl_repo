腾讯的前端面试中，算法考察的模式**介于“力扣（LeetCode）模式”和“ACM模式”之间，且更倾向于一种‘白板编程’的混合体验**。

简单来说：**题目逻辑像力扣，但环境可能像记事本，通常需要你自己手动写调用来验证。**

以下是详细的考察方式分析，以及你需要做的准备：

### 1. 考察平台与环境

腾讯面试目前绝大多数使用 **腾讯会议 (Tencent Meeting)** 内置的代码编辑器，或者面试官会发一个第三方链接（如牛客网、ShowMeBug，或者即时协作文档）。

- **腾讯会议代码面板（最常见）：**
    
    - 这更像一个简单的在线 IDE。
        
    - **没有自动补全**（或者很弱），没有像 VS Code 那么强大的提示。
        
    - **通常支持运行**，你可以选择 JavaScript (Node.js) 环境来执行代码。
        

### 2. 是“核心代码模式”还是“ACM模式”？

这取决于面试官使用的具体平台，但作为前端，你必须准备好应对**ACM 模式（需要自己处理输入输出）**。

- **力扣模式（Core Code Mode）：**
    
    - _场景_：面试官只给你一个函数壳子 `function solve(nums) { ... }`。
        
    - _要求_：你只需要补全函数体，返回结果即可。
        
    - _概率_：**约占 40%**。通常出现在面试官直接口述题目，或者使用特定的 OJ 平台时。
        
- **ACM 模式 / 白板模式（常见）：**
    
    - _场景_：编辑器是空的，或者面试官让你在记事本/飞书文档里写。
        
    - _要求_：你需要**自己定义变量**，**自己写函数**，并且**自己写测试用例来调用这个函数**并打印结果 (`console.log`)。
        
    - _概率_：**约占 60%**。
        
    - _注意_：如果是牛客网链接，通常需要处理 `readline` (NodeJS) 来读取标准输入（stdin），这对于只刷力扣的前端来说是一个**大坑**，一定要提前练习。
        

### 3. 需要自己写测试用例吗？

**需要，而且非常重要。**

即使是力扣模式，腾讯面试官也非常看重你的**测试意识**。

- **如果是不能运行的白板：** 你需要“人肉运行”（Dry Run）。你需要口头带着面试官走一遍代码：“假设输入是 `[1, 2, 3]`，第一步 i=0，...”。
    
- **如果是可以运行的 IDE：** 写完算法后，**不要等待面试官发问**，请自觉在代码最下方写上：
    
    JavaScript
    
    ```
    // 测试用例
    const input1 = [2, 7, 11, 15];
    const target1 = 9;
    console.log(twoSum(input1, target1)); // 预期输出 [0, 1]
    
    // 边界测试（加分项！）
    const input2 = [];
    console.log(twoSum(input2, 9)); // 预期输出 []
    ```
    
    _主动写出边界条件（空数组、极大值、负数等）的测试用例，是资深工程师的体现。_
    

### 4. 腾讯前端考什么类型的算法？

腾讯前端的算法题通常分为两类，你需要分开准备：

#### A类：标准算法题（数据结构与算法）

- **难度**：通常是力扣 **Easy** 到 **Medium** 级别，偶尔有 Hard（较少）。
    
- **高频题**：
    
    - 字符串操作（反转、无重复最长子串）。
        
    - 数组（两数之和、三数之和、合并区间）。
        
    - 树（层序遍历、最近公共祖先）。
        
    - 排序（快速排序，**必须能手写**）。
        
    - 动态规划（爬楼梯、最大子序和）。
        

#### B类：JavaScript 手写题（前端特有）

**这一部分在腾讯面试中占比非常高**，甚至超过纯算法题。这类题目不需要处理 ACM 输入，但需要严格符合 JS 语法。

- **高频题**：
    
    - 手写 `Promise.all` / `Promise.race`。
        
    - 手写防抖（Debounce）和节流（Throttle）。
        
    - 手写深拷贝（Deep Clone，考虑循环引用）。
        
    - 手写 `EventBus` (发布订阅模式)。
        
    - 数组扁平化 (`flat`)。
        
    - 解析 URL 参数为对象。
        

### 总结与建议

1. **不要只依赖力扣的“点击提交”**：在本地 VS Code 或者记事本里练习写代码，习惯没有代码补全的日子。
    
2. **练习 ACM 模式的输入处理**：去牛客网找几道“输入输出”的练习题，学会如何在 Node.js 环境下用 `readline` 读取多行输入（虽然前端考得少，但一旦考到不会写 `readline` 就直接挂了）。
    
3. **主动展示测试能力**：写完代码后，主动说：“我可以写几个测试用例跑一下吗？”
    
4. **关注边界条件**：腾讯面试官非常喜欢问“如果输入是 `null` 怎么办？”“如果数组长度是一百万怎么办？”




为了让你在练习时最大程度地还原腾讯（以及大多数大厂）的面试场景，我建议你把日常的“刷题模式”调整为**“模拟面试模式”**。

具体来说，你可以按照以下 **4 个步骤** 来构建你的训练流程：

### 第一步：“降维”你的编程环境（模拟白板）

面试中最让候选人恐慌的往往不是题目难，而是**没有代码提示**和**没有自动补全**。

- **平时练习：** 坚决**不要**在 LeetCode 网页编辑器里直接写，也不要用配置了 Copilot 的 VS Code。
    
- **模拟场景：**
    
    - 打开一个**空白的记事本**、Sublime Text（未安装插件版）或者在线的**腾讯文档/飞书文档**。
        
    - **手写所有代码**：包括 `function`, `return`, `console.log`，甚至括号匹配。
        
    - **目的：** 强迫自己记清楚 API 的完整拼写（比如是 `Array.prototype.splice` 还是 `slice`？参数顺序是什么？）。
        

### 第二步：背诵并熟练使用“输入输出模板”（针对 ACM 模式）

既然腾讯有时候会考 ACM 模式，你就必须把 Node.js 读取标准输入的“样板代码”背得滚瓜烂熟。不要等到面试时才去查怎么读多行数据。

**建议你把这套模板保存在本地，或者每天默写一遍：**

JavaScript

```
// Node.js 读取输入的通用模板 (背诵！)
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

const lines = [];
rl.on('line', (line) => {
  // 读取每一行输入，存入数组
  if (line.trim() !== '') {
    lines.push(line.trim());
  }
});

rl.on('close', () => {
  // 在这里处理所有输入
  if (lines.length === 0) return;

  // 举例：第一行是 N，后面是数组
  const n = parseInt(lines[0]);
  const arr = lines[1].split(' ').map(Number);

  // 调用你的核心算法函数
  const result = solve(n, arr);
  
  // 输出结果
  console.log(result);
});

// 你的算法函数
function solve(n, arr) {
  // ...
}
```

### 第三步：专项突击“手写 JS 实现”（前端必考）

腾讯非常喜欢考“造轮子”。这些题在 LeetCode 上很难找到原题，需要专门收集。建议你建立一个**“手写代码库”**，每天随机抽 2 个默写。

**必练清单（按优先级排序）：**

1. **异步编程：** `Promise.all`, `Promise.race`, `Promise.retry` (带重试功能的请求), 手写简易版 `Promise` (constructor, then, resolve)。
    
2. **功能函数：** 防抖 (`debounce`)、节流 (`throttle`)、深拷贝 (`deepClone`, 必须处理循环引用 `WeakMap`)。
    
3. **数组/对象：** 数组扁平化 (`flat`), 数组去重, `call/apply/bind` 实现, `new` 操作符原理。
    
4. **场景题：** 发布订阅模式 (`EventEmitter`), 图片懒加载逻辑, 解析 URL 参数 (`query string` 转对象)。
    

### 第四步：练习“自言自语”和“人肉测试”

这是面试能不能过的关键。面试官需要的不是一个沉默的打字机，而是一个能沟通的工程师。

- **练习步骤：**
    
    1. **读题后先不动手**：大声说出你的思路。“这道题我打算用双指针，左指针从 0 开始……”（假装旁边坐着面试官）。
        
    2. **写代码时解释**：“这里我判断 `!root` 是为了处理边界情况……”
        
    3. **写完后不要点运行**：先用肉眼看代码（Dry Run）。自己带入一个简单的例子 `[1, 2, 3]`，一行行在脑子里跑，看看逻辑通不通。
        
    4. **最后再运行**：如果报错，不要慌，边调试边解释“哦，这里少了一个分号，导致了……”
        

### 总结你的备战策略

1. **工具**：用记事本写代码，戒断自动补全。
    
2. **输入**：熟练掌握 Node.js `readline` 处理输入。
    
3. **题库**：LeetCode Hot 100（算法） + 手写 JS 高频题（前端特供）。
    
4. **流程**：看题 -> 口述思路 -> 编写代码 -> 编写测试用例 -> 人肉跑一遍 -> 提交。



### 🟢 第一梯队：JS 手写题（重中之重）

**这是腾讯前端面试的一大特色。** 面试官往往会把“实现一个功能函数”作为算法题来考。这部分如果写不出来，大概率直接挂。

1. **Promise 系列 (⭐⭐⭐⭐⭐)**
    
    - 实现 `Promise.all`、`Promise.race`（极高频）。
        
    - 实现一个带有并发限制的调度器（Scheduler，例如：同时最多发 2 个请求）。
        
    - 手写简易版 `Promise`（构造函数、`then`、`resolve`）。
        
2. **功能函数系列 (⭐⭐⭐⭐⭐)**
    
    - **防抖 (debounce) & 节流 (throttle)**：必须能手写，且要理解区别。
        
    - **深拷贝 (deepClone)**：要能处理循环引用（用 `WeakMap`），别只写 `JSON.parse`。
        
    - **数组扁平化 (flat)**：指定深度的扁平化（递归或 reduce）。
        
    - **手写 call / apply / bind**：理解 `this` 指向的核心。
        
3. **设计模式 (⭐⭐⭐⭐)**
    
    - **发布订阅模式 (EventEmitter)**：实现 `on`, `emit`, `off`, `once`。

### 🟡 第二梯队：高频经典算法（LeetCode Hot 100）

这部分是标准的算法题，主要考察逻辑思维。难度通常在 **Easy 到 Medium** 之间。

1. **字符串与数组 (⭐⭐⭐⭐⭐)**
    
    - **大数相加**：因为 JS 的 `Number` 有精度限制，这道题非常贴合前端场景（必会）。
        
    - **无重复字符的最长子串**（滑动窗口经典题）。
        
    - **两数之和 / 三数之和**。
        
    - **合并两个有序数组**。
        
    - **有效的括号**（栈的应用）。
        
2. **树 (Tree) (⭐⭐⭐⭐⭐)**
    
    - 前端 DOM 树就是树结构，所以树的考察非常多。
        
    - **层序遍历**（BFS）：输出二维数组。
        
    - **前中后序遍历**（递归 & 非递归）。
        
    - **二叉树的最大深度**。
        
    - **最近公共祖先**。
        
3. **排序算法 (⭐⭐⭐⭐)**
    
    - **快速排序 (Quick Sort)**：必须能**默写**代码，不仅是讲原理。
        
    - 归并排序：理解思想即可。
        
    - 第 K 个最大元素（快排变种）。
        
4. **链表 (⭐⭐⭐)**
    
    - 反转链表（迭代法 & 递归法）。
        
    - 环形链表（快慢指针）。
        
    - 合并两个有序链表。

### 🚀 备考策略建议

1. **“二八定律”复习法**：
    
    - 腾讯面试中，**80% 的时间**应该花在 **JS 手写题** 和 **Top 50 的通用算法**（如快排、二叉树遍历、大数相加）上。
        
    - 不要去死磕 Hard 级别的动态规划或图论，性价比极低。
        
2. **关注边界条件与鲁棒性**：
    
    - 腾讯面试官很喜欢问：“如果我传入的参数是 `null` 怎么办？”
        
    - 写代码时，开头先做参数校验。
        
3. **不要只刷题，要“背代码”**：
    
    - 对于 `Promise.all`、`快速排序`、`深拷贝` 这种死题，要练到肌肉记忆，提笔就能写，不要在面试时现场推导。
